# 🎨 DESIGN BRIEF — 삼국지 카드 게임 UI 리디자인

> 이 문서는 화면 리디자인을 AI/디자이너에게 맡길 때 **컨셉 · 기술 컨트랙트 · 레퍼런스**를
> 한 번에 전달하기 위한 브리프입니다. 작업은 **시스템 먼저 → 화면별**로 진행합니다.
> 첫 적용 화면: **첫 화면(랜딩, `index.html`)**.

---

## 0. 한 줄 컨셉
**삼국지 난세의 전장**을 무대로 한 **카드 로그라이크 + 보드 카드 대전** 모바일 게임.
무드 키워드 5: **묵직한 · 고풍스러운 · 금빛 · 전장의 긴박함 · 한 손 모바일**.

## 1. 아트 디렉션 (확정)
- **방향: 다크 로그라이크** (레퍼런스: *Slay the Spire*의 어둡고 거친 질감 · 맵/전투 가독성 중심).
- **에셋 정책: 인라인 SVG + CSS만** — **외부 파일 0개**(이미지/폰트/아이콘 라이브러리 금지).
  아이콘·프레임·문양·노이즈·그라데이션은 **인라인 `<svg>` 또는 CSS**로 구현. 이모지는 보조로 유지 가능.
- 동양 사극 모티프(두루마리·인장·먹·금박·죽간)를 **다크 로그라이크 질감 위에** 절제해 얹는다.
- 무의존(빌드 도구 없음) 유지: 순수 HTML/CSS/JS.

## 2. 절대 제약 — "컨트랙트" (깨면 게임이 멈춤)
리디자인은 아래를 **반드시 유지**하거나, 바꾸려면 **해당 JS도 함께 수정**해야 합니다.

1. **DOM 훅 유지**: JS가 `id`/`class`/`data-*`로 요소를 찾습니다. 주요 훅:
   - 공통: `#fxLayer`, `.screen`, `.modal`, `.overlay`, `.btn`, `.btn.primary`, `.btn.ghost`, `.lang-bar`/`.lang-btn`
   - 영웅전 지도: `#mapTitle` `#mapSub` `#mapTrack` `.main-chips` `.main-chip` `.stage-node`(`.normal/.mid/.boss/.done/.current/.locked/.info`) `.sub-dots` `.main-banner` `.camp-btn[data-act]` `.map-lord-status` `.map-collect`
   - 전투: `#enemyRow` `.unit`(`.u-tag.boss/.mid`) `#combatLog` `#lordBar` `.combat-board` `.draw-pile/.center-cards/.used-pile` `.action-bar` `.act-btn[data-act]` `#itemBar` `#combatHint` `#endTurnBtn`
   - 보상/상점/주막/이벤트: `#rewardCards` `.reward-card` `#shopItems` `#tavernItems` `#eventChoices`
   - 도감/모달: `#codexModal` `.codex-tab[data-tab]` `.col-grid` `.col-card` `.col-weapon-list` `.gear-row` `.sort-btn[data-sort]` `#heroModalBody`
   - 히어로즈 블러드: `#board` `.cell` `#hand` `.qb-card` `.player-strip` `#turnInd` `#passBtn` `#scoreboard`
2. **다국어(i18n) 유지**: 정적 문구는 `data-i18n="키"`, 동적 문구는 JS `TCG.t('키')`.
   - 새 텍스트 추가 시 `js/i18n.js`(정적) 또는 `js/i18n_data.js`의 `__UI_I18N__`(동적)에 **5개 언어**(ko/en/ja/zh/zhTW) 등록.
   - 마크업 변경 시 `data-i18n` 속성과 텍스트 노드 위치를 보존(라벨 안에 동적 `<b id=...>`가 끼어있는 패턴 주의).
3. **모바일 세로 우선**: 한 손/엄지 영역. `dvh` 기준, 노치/주소창 대응. 데스크톱은 중앙 정렬 보조.
4. **배포 규칙 유지**(`DEPLOY.md`): 배포마다 마이너 버전 +1, 에셋 `?v=` 캐시버스터, 첫 화면 변경 내역 갱신. CSS만 바꿔도 동일.
5. **단일 빌드**(`dist/play.html`)는 `scripts/build-single.js`가 인라인 번들 — CSS/HTML 구조 변경 후 재생성 필요.

## 3. 디자인 토큰 (현재값 — 시스템화 대상)
`css/common.css :root`:
```
--bg-0 #140d0b   --bg-1 #1f1411   --bg-2 #2c1b15      /* 먹빛 배경 3단 */
--panel rgba(48,30,24,.74)   --panel-solid #2a1a14    /* 패널 */
--gold #f0c33c   --gold-soft #cf9a2a                  /* 금 (강조/테두리) */
--ink #f3e9da    --ink-dim #c4ab90                    /* 본문/보조 텍스트 */
--red/--foe #e0483f   --green #5bb87a   --blue/--you #4f9bff
--radius 14px    --shadow 0 8px 24px rgba(0,0,0,.5)
```
- 진영색: 영웅전 `#e0483f`(주홍) · 대장전 `#9b5de5`(보라) · 블러드 `#f0c33c`(금).
- **요청**: 위 토큰을 출발점으로 **다크 로그라이크 팔레트로 확장**(표면 단계·테두리/광택·상태색·등급색 C/R/SR/SSR·치명타/독/방어 등 시맨틱 컬러), **타이포 스케일·간격·라운드·그림자·테두리 프레임 SVG**를 한 시스템으로 정의.

## 4. 화면 인벤토리
### index.html — 첫 화면(랜딩) ★ 1순위
- 목적: 3개 게임 진입 + 정체성 각인. 핵심 행동: **게임 카드 탭**.
- 요소: 게임 카드 3종(`.game-card.heroes/.raid/.queens` — 제목/태그/설명/플레이 필), 하단 버전 뱃지·가이드/변경내역/만든이·**언어 선택 바**, 모달 3종.
### heroes.html — 삼국 영웅전(메인)
- **지도/대기실**(`#mapScreen`): 연대기 진행(8전역 칩 + 11출진 스테이지 노드), 대기실 액션(출진/진형/상점/주막), 주공 HP/MP/추가능력.
- **전투**(`#combatScreen`): 적 행렬, 전투 로그, 주공 바, 카드 3분할(뽑을/가운데/사용), 액션 바, 아이템 바, 힌트/턴종료.
- 보상/휴식/상점/주막/보물 이벤트 화면, 시작·도감·장수상세·골드팝업 모달.
### daejang.html — 삼국 대장전(레이드)
- 레이드 선택 리스트, 전투(보스+졸병 다중 타겟), 덱 편성·장착 장비·도감·보스 정보 모달.
### queensblood.html — 히어로즈 블러드(보드 카드 대전)
- 보드(3×5/4×5), 손패, 점수판, 플레이어 스트립, 턴바, 규칙·덱빌더·카드정보·결과 모달.

## 5. 컴포넌트 인벤토리 (시스템으로 통일)
카드(전투/보드/보상/도감) · 버튼(primary/ghost/act/sort) · 패널·모달 · HP/MP 바 · **스테이지 맵 노드/경로**(`.stage-node`) · 적/아군 유닛(`.unit`) · 태그/배지(등급·적장·중간보스) · 토스트/배너/플로팅텍스트 · 언어 선택 바.
→ **각 컴포넌트의 인라인 SVG 프레임/아이콘 1세트**를 먼저 정의하면 전 화면에서 재사용됨.

## 6. 작업 방법 (권장)
1. **디자인 시스템 PR**: 토큰 + 카드 1종 + 버튼 1종 + 패널/모달 1종 + 핵심 SVG 아이콘/프레임 세트. (실제 화면 1개에 데모)
2. **랜딩 적용**: 시스템으로 `index.html` 리디자인(아래 7장).
3. **화면별 순차**: 지도 → 전투 → 보상/상점 → 결과 → 대장전 → 블러드.
4. **반복**: 각 화면 **전/후 스크린샷**으로 피드백. (현재 화면은 https://gnsyoo.github.io/tkh-tcg/ 에서 캡처)

## 7. 랜딩(첫 화면) 상세 브리프 ★
- **현재 상태/문제**: 게임 카드 3개 + 하단 텍스트 푸터로 다소 밋밋. "게임 타이틀 화면" 같은 압도감/세계관 각인이 약함.
- **목표 감정**: 진입 즉시 "삼국지 다크 카드게임"임을 각인. 카드 3종이 **선택지로서 매력적**으로 보이게.
- **핵심**: ① 상단 타이틀/엠블럼(인라인 SVG 문양 + 금박) ② 게임 카드 3종을 **진영색·아이콘·질감**으로 차별화(주홍/보라/금) ③ 진입 동작 강조(플레이 필) ④ 하단 유틸(버전·가이드·**언어 바**)은 조용히.
- **유지**: 카드 링크(`#cardHeroes/#cardRaid/#cardQueens` → 각 html), `data-i18n` 텍스트, 언어 바(`#langBar`/`.lang-btn[data-lang]`), 모달 트리거(`#guideBtn/#changelogBtn/#creditsBtn`), 버전 뱃지(`#verBadge`, release 스크립트가 자동 갱신).
- **레퍼런스**: 다크 로그라이크 메인 메뉴(어두운 배경 + 강한 포컬 + 거친 텍스처), 삼국지 SLG의 금속/두루마리 프레임.

## 9. 산출물 & 반복 형식
- 산출물: **CSS(+인라인 SVG) 코드**로 직접 적용(목업 이미지가 아니라 실제 화면). 컴포넌트/화면 단위 커밋.
- 제약 재확인: 외부 파일 0 · DOM 훅/`data-i18n` 보존 · 모바일 세로 · 배포/버전 규칙.
- 변경 후: `node scripts/build-single.js`로 단일 빌드 갱신, `node scripts/release.js`로 배포.
